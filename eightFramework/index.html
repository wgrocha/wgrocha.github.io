O que é o eightFramework ?

O eightFramework trata-se de um framework simples e leve para desenvolvimento de páginas web responsivas, criado pensando-se em portabilidade, legibilidade e simplicidade, com o objetivo de apresentar o conteúdo de forma consistente em diferentes tipos de dispositivos. O projeto surgiu a partir de estudos sobre HTML5 e CSS3 como uma maneira de aplicar os conhecimentos adquiridos. Portanto não se trata de um framework completo com centenas de funcionalidades, ainda assim faço uso frequente desta suíte de regras CSS para desenvolvimento de aplicações web mais complexas, realizando apenas alguns ajustes e fazendo uso conjunto com outras ferramentas."

Willian Gabriel Cerqueira da Rocha

Funcionamento e utilização

Ícones prontos para uso

Uma coleção de ícones simples e eficiente, são 290 ícones em formato de fonte e mais 312 em formato de imagem.

Fácil de usar

A facilidade de utilização melhora o desempenho e a qualidade do trabalho, basta atribuir classes aos elementos html e é possível criar interfaces fluidas, responsivas e bem estruturadas.

Responsivo

São as classes dos elementos html que definem como a página será renderizada em diferentes dispositivos. Através de regras simples é possível definir essa visualização de maneira eficiente.

Leve

Prático e minimalista, o Eight Framework é extremamente leve, ocupando apenas alguns KB's de espaço em disco.

 Sistema de colunas

Para utilizar os recursos do framework basta incluir os arquivos eight.css, default-theme.css e eight.js no header da página: 

<link rel="stylesheet" href="css/eight.css">
<link rel="stylesheet" href="css/default-theme.css">
<script src="js/eight.js"></script>

O arquivo default-theme.css tem as principais propriedades que podem ser alteradas sem causar problemas no eightFramework, podendo servir portanto como base para criação de diferentes temas para o framework.

OBS.: Para o bom funcionamento dos recursos do Eight Framework faz-se necessário o uso da seguinte <meta> tag: 

<meta name="viewport" content="width=device-width, initial-scale=1">

Leia mais sobre isso aqui.



Antes de tudo cabe esclarecer a utilização de containers e da classe full-viewport, o viewport é a área visível que o usuário tem da tela, é possível configurar a página para que ela seja exibida em toda a largura disponível, ou configurar para que apenas seções específicas sejam exibidas assim.


Para isso, adiciona-se a classe full-viewport ao elemento em questão, esta classe aplica-se às tags body, section, header e footer.


Se aplicada à tag body, todas as seções do corpo da página ocuparão a largura completa da tela do usuário, independente do tamanho da tela; se aplicada à tags to tipo section, header ou footer apenas aquela tag ocupará o viewport horizontal completo.


No caso de omissão desta classe, será deixado um espaço vazio dos lados do corpo da página, o conteúdo do body será centralizado na tela, ficando uma margem à direita e uma margem à esquerda. Nos dispositivos do tipo small e extra-small não há diferença entre usar ou não full-viewport, o conteúdo será apresentado em toda a largura disponível da tela destes dispositivo.


É possível ainda, desejar que um elemento ocupe toda a largura disponível da tela, mas que os elementos internos ocupem apenas a porção central. O eightFramework utiliza uma divisão da tela em 12 colunas. Para especificar como será feita a apresentação do conteúdo, o programador define quantas colunas serão ocupadas pelos elementos em determinado tipo de dispositivo. É através da atribuição de classes aos elementos html que define-se de que maneira o conteúdo será apresentado. As classes que definem estas características são as seguintes:

xlarge-X large-X medium-X small-X xsmall-X

Diferentes dispositivos definidos pelo eightFramework
Tipo de dispositivo Largura da tela (px)
Extra Large ≥ 1600
Large 1200 ~ 1599
Medium 990 ~ 1199
Small 768 ~ 989
Extra small ≤ 767

Por exemplo, se o programador deseja que determinada div seja apresentada em 1/4 da tela quando o dispositivo for do tipo Large ele deve atribuir a esta div a classe large-3, o que indica que a div irá ocupar três colunas das 12 definidas pelo framework.

<div class="row">
	<div class="large-3">Some content</div>
</div>

Caso não seja especificado para um elemento quantas colunas ocupar em determinado dispositivo, ele ocupará automaticamente todas as 12 colunas, note que no exemplo anterior não foi especificado que o elemento deveria ocupar toda a tela para dispositivos diferentes de large, mas esse comportamento é automático.

Caso deseje-se que o elemento ocupe toda a largura da tela independente da situação, deve-se atribuir a ele ao menos uma das classes (xlarge-12, large-12, medium-12, small-12 ou xsmall-12).

O sistema de colunas é similar às tableas do html, define-se linha (row) e colunas (xlarge-X, large-X, medium-X, small-X e xsmall-X) com a diferença de que usando esse sistema o layout pode ser projetado de maneira eficiente e responsiva. Em alguns casos o uso de colunas fora de linhas pode apresentar resultados totalmente satisfatórios e bem formatados, tanto que pode-se ter visualmente múltiplas linhas apenas com a declaração de colunas (quebra automática), mas é recomendável que sempre utilize-se colunas dentro de linhas, como foi feito no exemplo anterior.

Neste outro exemplo são definidas duas colunas que ocupam a mesma linha em telas de tamanho xlarge, large, medium e small. Note que os valores para cada tipo de dispositivo são complementares, a soma deles resulta em 12. Caso algum desses valores ultrapasse o número de colunas disponíveis na linha, a coluna será apresentada na próxima linha (quebra automática). 

<div class="row">
	<div class="xlarge-2 large-2 medium-4 small-6">Div1</div>
	<div class="xlarge-10 large-10 medium-8 small-6">Div2</div>
</div>

Como já dito anteriormente, em caso de omissão, o elemento ocupará todas as 12 colunas, é exatamente o que acontecerá caso o conteúdo deste exemplo seja apresentado em telas de tamanho extra small.

Deslocamento de colunas

Pode haver algum caso em que seja necessário deslocar colunas que não ocupam todo o espaço disponível da linha, para isso existem as seguintes classes, em que X é o número de colunas a deslocar para a direita:


  • xlarge-offset-X
  • large-offset-X
  • medium-offset-X
  • small-offset-X
  • xsmall-offset-X

<div class="row v-align-middle">
	<div class="xlarge-4">
		Some content
	</div>
	<div class="xlarge-4 xlarge-offset-5">
		More content
	</div>
</div>

Repare que no exemplo anterior sobrou um espaço de 4 colunas entre as duas divs definidas, isso ocorre porque houve um desolcamento para a direita; em telas xlarge o conteúdo da segunda div irá iniciar na quinta coluna a contar do final da primeira div.

Ocultando elementos seletivamente

É possível ocultar elementos a depender do tipo de dispositivo que estiver sendo utilizado.

Isso é feito com a atribuição das seguintes classes:


  • xlarge-hide  Oculta o elemento em telas do tipo xlarge
  • large-hide  Oculta o elemento em telas do tipo large
  • medium-hide  Oculta o elemento em telas do tipo medium
  • small-hide  Oculta o elemento em telas do tipo small
  • xsmall-hide  Oculta o elemento em telas do tipo xsmall

 Ícones

O eightFramework conta com uma coleção de 290 ícones em formato True Type e Woff. Estes ícones foram obtidos em icomoon.io. Eles podem ser utilizados nos formulários, nas listas e de maneira independente em qualquer parte da página. Nesta seção será explicado apenas o uso de ícones em partes independentes da página; para saber como usar em formulários e listas siga para as seções específicas.

Para utilizar ícones é recomendado o uso da tag span, apesar de que outras tags como p, div, i, b, entre outras, também funcionam normalmente.
Veja o exemplo a seguir: 


<span id="myIcon" class="icon icon-home"></span><!--Sintaxe recomendada-->
<span class="icon">icon-home</span><!-- Ligature -->

No segundo caso está sendo usado um recurso denominado ligature, compatível apenas com navegadores mais modernos, por isso é recomendável a primeira sintaxe. Como os ícones estão em formato de fonte então é possível customizá-los com uso de css da maneira convencional: 

#icon-example {
	font-size: 100px;
	background-color: #0af;
	color: #555;
	padding: 10px;
	border: 3px solid #555;
	border-radius: 15px;
}

 Tabelas

Tabelas podem ser utilizadas da maneira padrão pois as únicas modificações que o eightFramework aplica a elas são em estilo, não há modificações na lógica, estrutura ou usabilidade. São definidas apenas duas classes para as tabelas, são elas rounded-borders e cell-borders, elas podem ser utilizadas idividualmente ou em conjunto; rounded-borders aplica bordas arredondadas à tabela e cell-borders adiciona bordas internas (que por padrão estão desativadas): 

<table class="cell-borders rounded-borders">
	<caption>An example</caption>
	<thead>
		<tr>
			<th>first column header</th>
			<th>second column header</th>
			<th>third column header</th>
		</tr>
	</thead>
	<tbody>
		<tr>
			<td>1.1 data</td>
			<td>1.2 data</td>
			<td>1.3 data</td>
		</tr>
		<tr>
			<td>2.1 data</td>
			<td>2.2 data</td>
			<td>2.3 data</td>
		</tr>
	</tbody>
</table>

 Mobile

É possível fazer uso de um menu de navegação colapsável, muito útil para telas pequenas e dispositivos móveis.

Para uilizá-lo é necessário definir a seguinte estrutura: 


<header class="navigation-header fixed-top">
	<div class="row v-align-middle">
		<div class="xlarge-4 xlarge-offset-2 large-4 large-offset-2 medium-5 medium-offset-2 small-10 xsmall-9">
			<h1>Places to visit</h1>
		</div>
		<nav class="collapsable-nav xlarge-4 xlarge-offset-3 large-4 large-offset-3 medium-5 small-2 xsmall-3">
			<ul>
				<li><a href="#streets">Streets</a></li>
				<li><a href="#bridges">Bridges</a></li>
				<li><a href="#markets">Markets</a></li>
				<li><a href="#restaurants">Restaurants</a></li>
				<li><a href="#schools">Schools</a></li>
			</ul>
		</nav>
	</div>
</header>

O essencial para que o menu funcione é que a classe navigation-header seja atribuída ao header e que exista uma tag nav dentro deste header. A essa tag nav deve ser atribuída a classe collapsable-nav, dentro desta nav deve-se então definir uma lista com os links que serão utilizados no menu de navegação.

O menu irá colapsar em telas de tamanho small e xsmall.

Ao atribuir a classe navigation-header ao header, ele será customizado de acordo com o padrão do framework. Caso deseje-se utilizá-lo para outra finalidade que não seja um menu de navegação, evite o uso da classe navigation-header.

Para fixar o menu no topo da janela do browser é preciso atribuir a classe fixed-top a ele, assim ele ficará visível todo o tempo enquanto o usuário navega pela página. Há de se destacar que quando o header possuir a classe fixed-top ele deixará de fazer parte do fluxo padrão da página, então será necessário configurar uma margem para a próxima seção do documento (que corresponda à altura ocupada pelo header).

Para uma boa formatação, deve-se definir apenas uma row dentro do header e a tag nav deve ser filha imediata desta row. A tag nav também deve representar colunas desta row.

 Listas

Assim como as tabelas, as listas não possuem muita diferença da formatação e comportamento padrão, a única diferença é que o framework permite a atribuição de ícones aos elementos de listas não ordenadas. Isso pode ser feito atribuido a classe ico-list à tag ul e atribuindo então ícones para os itens da lista. Veja o exemplo: 

<ul class="ico-list">
	<li class="icon-search">Item1</li>
	<li class="icon-instagram">Item2</li>
	<li class="icon-google3">Item3</li>
	<li class="icon-facebook">Item4</li>
	<li class="icon-vimeo">Item5</li>
</ul>

A atribuição de uma classe icon-{something} para cada elemento da lista serve para especificar o ícone que será utilizado para aquele item, note que caso não seja atribuída a classe ico-list à tag ul, os ícones não serão renderizados. Veja aqui a lista completa de ícones disponíveis.

 Formulários

Os elementos mais comuns de formulários contam com formatação de estilos aplicada pelo eightFramework com o objetivo de assimilar um design particular a eles, veja o exemplo: 

<form>
	<fieldset>
		<legend>Login</legend>
		<p><label for="username">Usuário: </label><input type="text"name="username"></p>
		<p><label for="userpass">Senha: </label><input type="password" name="userpass"></p>
		<p><input class="blue-button" type="submit" name="send" value="Send" onclick="return;"></p>
	</fieldset>
</form>
Login

Como pode-se notar, neste exemplo há a atribuição da classe blue-button ao botão de submissão do formulário, o eightFramework define dois tipos de botões, são eles blue-button e green-button, como o nome sugere, a diferença entre eles é apenas a cor. Essas classes são aplicáveis a inputs do tipo submit e reset e a elementos do tipo button.

Por padrão o fieldset é renderizado com uma borda cinza claro, para remover basta atribuir a ele a classe no-border.

Também é possível utilizar ícones dentro dos inputs, para isso é necessário usar uma tag como container e atribuir a esta tag a classe ico-input e tabém a classe correspondente ao ícone, veja o exemplo, agora com ícones e sem bordas no fieldset: 

<form>
	<fieldset class="no-border">
		<legend>Login</legend>
		<p><label for="username">Usuário: </label><span class="ico-input icon-user"><input type="text" name="username"></span></p>
		<p><label for="userpass">Senha: </label><span class="ico-input icon-lock"><input type="password" name="userpass"></span></p>
		<p><input class="green-button" type="submit" name="send" value="Send" onclick="return;"></p>
	</fieldset>
</form>
Login

Para customizar os ícones deve-se alterar as propriedades da tag que está servindo de container para o input.
